Delegating to a worker, the cage

delegate→ organ-graph check→ gate the task→ run caged→ return + propose→ brain stores

A worker never holds a raw adapter. Inside the cage its tool calls are checked against its own allowlist and the gate, its model calls pass DLP both ways, and it sees only its memory slice. It returns a result and proposes what it learned, the brain, not the worker, decides what is stored (secret-bearing learnings are refused). Learning drains inward: the worker's private store is never the canonical home.

The seven rules no adapter may break

Fail closed
A gate that can't decide denies. A broken guard stops the flow.
Own the data
The memory ledger is ours, on-box, append-only. No external store is the source of truth.
Govern on the port
Every call passes the gate at the contract boundary. Adapters never self-govern.
Tighten only
Automatic actors can only make things stricter; loosening is a human act with a receipt.
Re-gate on change
A scheduled or looped action is re-checked at fire time against current calibration.
Nothing lands ungoverned
New models, tools, or a worker's skills pass quarantine → scan → register → sandbox.
Learning drains inward
What a worker learns is proposed to the brain and stored in our ledger, or not at all.
